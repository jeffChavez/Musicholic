Detailed Description

Logging options specifies what options are applied to the GNSDK log

Method Documentation

- (GnLogOptions*) archive: (BOOL)  bArchive

Specify for the archive to retain and rename old logs, default behavior is to delete old logs

- (GnLogOptions*) archiveDaily

Specify that when archive is also specified the logs to archive (roll) at the start of each day (12:00 midnight). Archiving by the given size parameter will still occur normally as well.

- (GnLogOptions*) maxSize: (NSUInteger)  maxSize

Specify the maximum size of log before new log is created. Enter a value of zero (0) to always create new log on open

- (GnLogOptions*) synchronous: (BOOL)  bSyncWrite

Specify the log to be written synchronously (on the same thread). By default logs are written to asynchronously. No internal logging thread is created if all GnLog instances are specified for synchronous writing.
