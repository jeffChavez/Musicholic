Modules Overview

GNSDK for Mobile consists of several modules that support specific Gracenote products. The principal module required for all applications is the GNSDK Manager. Others are optional, depending on the functionality of the applications you develop and the products you license from Gracenote.

General GNSDK for Mobile Modules

GNSDK Manager: Required. GNSDK Manager provides core functionality necessary to all Gracenote modules.

Contact your Gracenote Global Services & Support representative for more information.

SQLite: . The SQLite module provides a local storage solution for GNSDK for Mobile using the SQLite database engine. This module is primarily used to manage a local cache of queries and content that the GNSDK for Mobile modules make to Gracenote Service. SQLite is a software module that implements a self-contained, server-less, zero-configuration, transactional SQL database engine. SQLite is the most widely deployed SQL database engine in the world. The source code for SQLite is in the public domain. Information on SQLite can be found at http://www.sqlite.org.

Product-Specific Modules

MusicID: Provides CD TOCTable of Contents. An area on CDs, DVDs, and Blu-ray discs that describes the unique track layout of the disc., text, and fingerprint identification for music.

MusicID-File: Provides file-based music identification using LibraryID, AlbumID, and TrackID processing functionality.

MusicID-Stream: Designed specifically to recognize music delivered as a continuous stream. Use this library in applications that need to recognize streaming music in real time and on-demand.

PlaylistA set of tracks from a user’s music collection, generated according to the criteria and limits defined by a playlist generator.: Provides functionality to generate and manage playlists and playlist collections. Also supports creation of MoodGrids to visually group content based on mood metadata.
